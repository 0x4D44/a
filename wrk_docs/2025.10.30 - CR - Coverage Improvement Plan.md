# Coverage Improvement Plan - 30 Oct 2025

## Objective
Raise overall coverage to >=90% while keeping the CLI testable across platforms and without relying on external services.

## Stage Status
- Stage 1 – Strengthen CLI integration coverage: Completed via expanded smoke tests that exercise add/remove/list/which flows (including the new coverage probe command) with cross-platform command scaffolding.
- Stage 2 – AliasManager unit backfill: Completed by covering confirmation flows, chain execution helpers, and parameter substitution edge cases.
- Stage 3 – GitHub token discovery & sync paths: Completed with stubbed gh/git binaries, additional GitHub client scenarios (status errors, body fallback, invalid encoding), and refactoring to simplify error propagation.
- Stage 4 – Platform-specific execution helpers: Completed through shared helpers and tests that validate PATHEXT handling and command preparation on both Windows and non-Windows targets.
- Stage 5 – Regression guard & clean-up: Completed. Coverage (cargo llvm-cov) now reports 97.35% lines and 93.72% functions; test, fmt, and lint suites remain green, and documentation has been updated.

## Lessons Learned
- Explicit tests plus targeted refactors (removing map_err/ok_or_else) were necessary to eliminate synthetic uncovered functions.
- Providing stub binaries in integration tests keeps token and GitHub sync logic deterministic without network access.
- Sharing cross-platform command helpers prevented Windows-specific assumptions from leaking into Linux coverage runs.

## Remaining Ideas (post-90%)
1. Create a harness that drives the CLI through failure exits while capturing std::process::exit codes.
2. Inject failing writers/readers to execute the remaining I/O error branches in confirmation prompts and GitHub sync paths.
3. Extend PATHEXT scenarios (for example, separator-only values) to cover the last guard branches.

## Validation Checklist
- [x] Each stage executed with local coverage snapshots recorded.
- [x] No reliance on external network or real GitHub credentials in new tests.
- [x] Temp directories cleaned automatically (scoped TempDir).
- [x] Coverage summary reports >=90% line and function coverage (current: 97.35% lines, 93.72% functions).
